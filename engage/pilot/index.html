Cannabis Grow Pilots

3D canopy-zone mapping. Verification. Receipts you can operate on.

These pilots are built for one outcome: turn “we swear we held setpoints” into a measurable field with acceptance tests.

A One-Time Map is Cheaper

  • One scan = baseline pocket map.
  • Cost scales with repeat scans (lights, irrigation, dehu).
  • Verification is the work: before/after proof.
  • Pass/fail receipts make it defensible.

Canopy pockets are cycle-driven

  • Pockets show on triggers, not averages.
  • Short pilots: find + locate.
  • Long pilots: repeatability + recurrence.
  • Late-flower + night cycles matter most.

Why These Pilots Are Worth Doing

  • Pockets inside the canopy are common.
  • Powdery mildew pressure is an industry staple.
  • Botrytis (“bud rot”) can move fast under high RH.
  • Compliance failures (yeast/mold) and recalls are real.
  • Testing requirements vary by state (and can include Aspergillus targets).

What Every Pilot Includes

  • Commissioning & acceptance tests
  • Validity / trust layer in the portal
  • Run receipts
  • Pilot readout

Pricing Scales with

  • Scope of measurement geometry
  • Time horizon (cycles observed)
  • Verification burden (proving a fix works)
  • Deliverable sophistication (ops vs QA vs exec)
  • Stakeholder/support expectations (logins, handoffs, training)

How to Start

  1. Site call (30–45 min): goals, constraints, room count, and where you want decisions to be easier.
  2. Pilot proposal: success criteria, deliverables, acceptance tests, and timeline across cycles.
  3. Install + commissioning: go-live only after acceptance tests pass and validity is demonstrated.
