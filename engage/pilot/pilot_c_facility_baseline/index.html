Primary outcome

  • Variance report: why Room 2 behaves like a different planet
  • Templates: placement + acceptance tests that travel room-to-room
  • Rollout playbook: how to baseline new rooms without guessing

Deliverables

  • Per-room zone maps: pockets + gradients + cycle timing
  • Cross-room comparisons: common pocket archetypes and outliers
  • Facility baseline package: configs, receipts, validity uptime, change history
  • Operator-ready standard: what “good” looks like and how to verify it

Typical timeline

  1. Plan: select 2–4 rooms + define cycles to capture (lights/irrigation/night)
  2. Baseline: instrument each room + run commissioning acceptance tests
  3. Compare: quantify variance and identify repeatable failure geometries
  4. Standardize: ship templates + rollout playbook

Acceptance tests (examples)

  • Consistency: facility “good state” is measurable, not tribal knowledge.
  • Comparability: cross-room comparisons only when validity is met (otherwise abstain).
  • Transferability: placement and acceptance tests can be applied to a new room.

Why facilities buy this

  • The same airflow and humidity dynamics that drive PM/botrytis risk can vary room-to-room.
  • Standardization reduces regressions when staff change, SOPs drift, or equipment is swapped.
