What it does

Construction Selection answers the question dashboards usually hide: ‚ÄúWhich construction did you use‚Äîand why that one?‚Äù The system evaluates a controlled set of candidates and selects using a declared rule, leaving receipts.

Candidate set

A declared, controlled set of allowed constructions ùìú (models, frames, transforms, aggregations). No silent ‚Äútry everything until it looks good.‚Äù

Selection rule

A declared rule R (thresholds, constraints, multi-objective tradeoffs) that determines what ‚Äúbest‚Äù means before results exist.

Receipts

Candidate scores + rule inputs + the chosen construction are recorded so a second operator can reproduce the choice.

If candidates tie, diverge, or selection is unstable, the system can withhold selection or downgrade the claim instead of quietly picking the most flattering frame.

Why this matters

In real systems, different constructions can produce different ‚Äútruths‚Äù from the same data: different transforms, different coordinate frames, different filters, different model families. If selection is not explicit, the pipeline can become self-authorizing by silently choosing the construction that makes the story look best.

If you can change the frame, you can change the conclusion.

Construction Selection forces the choice to be declared, scored, and reproducible.
